UX Updates

- I should be able to select a task as my current focus (or otherwise start/pause it).
- Multistep tasks should start the next available subtask in the list
- I should be able to uncheck a task, in case i've checked it off by mistake
- I should be able to create a new category while creating a new task
- A task can have multiple categories
- I should be able to complete a task that has no subtasks. Completing subtasks should autocomplete a task the same as it does today
- A recurring task should show as complete until it's recurring date comes up. It should also have visible copy that says the date/time it will recur.
- header menu should remain visible on all views

Matrix

- I should be able to drag cards to a different section of the matrix to change their priority

Analytics

- Recurring tasks should appear in the analytics view. They may need to exist as instances of a recurring task template to keep track, but use whatever approach makes the most sense.

Sorting

- Priority should have tasks with earlier due dates at the top. If a task is recurring, and has the same priority and is incomplete, it should appear above tasks with a due date later than the current date.

Settings

- There should be a settings UI so it's easy to adjust settings.

Mobile

- Ensure app is usable on a mobile view, using common mobile app patterns like collapsing menu into a hamburger

Future

- Calendar Integration
  - Add to calendar button that let's you manually add a task to your calendar for the amount of time. It shouldn't need to hook into your calendar, just generate an event that you can add or open the event in google calendar etc
